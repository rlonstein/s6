How to upgrade s6

from 0.17 to 1.0.0

Version 1.3.0 or later of skalibs, as well as version 1.2.0 or later of execline, are needed to build and/or run s6.

FHS support has been added. See the end of this page for the rationale, and the general skarnet.org configuration page for how to proceed.
This means you can now install the s6 binaries wherever you want.
However, since s6 binaries sometimes call other s6 binaries, and non-slashpackage installations cannot guarantee absolute paths, s6 programs now rely on the PATH environment variable to find all their executables on non-slashpackage installations; in other words, if you are not extra careful with how you are setting up your PATH, things might break.

from 0.16 to 0.17

Version 1.2.8 or later of skalibs, as well as version 1.1.5 or later of execline, are needed to build and/or run s6.

from 0.14 or 0.15 to 0.16

Version 1.2.6 or later of skalibs is needed to build and/or run s6.

from 0.13 to 0.14 or 0.15

No change in dependencies.

from 0.12 to 0.13

Version 1.1.1 or later of skalibs is needed to build and/or run s6.

from 0.11 to 0.12

You are advised to use at least version 1.0.2 of skalibs, for build-time as well as run-time (if you depend on shared skalibs libraries), since older versions have a bug that impacts s6.

from 0.10 to 0.11

No change in dependencies.
